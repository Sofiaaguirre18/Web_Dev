LOCKER  MANAGEMENT SYSTEM
MANUAL
HOMEPAGE

           The Homepage serves as the entry point to the Locker Management System. Users can initiate transactions by entering their school ID and locker preference. It ensures registered users can smoothly proceed to avail lockers, while unregistered users are guided to log in first. The transaction form collects user information, allowing secure locker access. Additionally, the Homepage provides a quick glimpse of the last user's locker transaction details for added convenience.

TRANSACTION
  1. The user will first enter their school ID and what locker will they avail.
  2. The System will check if the school ID inputted was already registered in the login system, if not then a notification will pop up that the user must login first before proceeding to avail a locker.
  3. If the school ID provided by the user is already registered in the login system, then the system will direct the user to the transaction form where in he'll have to fill up the following information ( the school ID and locker is given ).
    • Name : Surname, First name M.I
    • Account type: Student/Employee
    • Department : What department is the student came from or what department the employee is assigned [P.S other option will also presented in case the employee is not part of any department]
    • Gender: Male, Female, others
  4. If the information is all completely filled up then the patron’s information will be stored in the database with a status of “pending”, however if the information given by the user is not yet filled up or some information is inappropriate then the system will ask the patron to input other valid input that correspond to what is being asked
